# AGENTS.md

## 项目定位

- 这是一个仅供个人在本地使用的 tmux 插件，不是公共库或面向外部用户的产品。
- 开发新功能时，以当前需求、代码清晰度和长期可维护性为优先。
- 不需要考虑向后兼容；不要为了旧配置、旧接口、旧行为或旧数据格式保留兼容分支。
- 如果更直接的设计会产生破坏性变更，可以直接修改，同时删除已经失效的实现、测试和文档。
- 不要添加兼容层、废弃期、迁移适配器、双写逻辑或新旧实现并存的兜底方案，除非任务明确要求。

## 编码原则

- 使用**显式**写法，不使用依赖隐含行为、隐藏副作用或难以推断上下文的写法。
- 变量、函数、参数和状态名称必须准确表达用途；不要使用含义模糊的缩写。
- 明确写出输入校验、默认值、状态转换、错误处理和返回路径。
- 优先使用容易顺序阅读的控制流；避免为了减少行数而使用复杂的短路表达式、嵌套三元表达式或过度紧凑的一行代码。
- TypeScript 中为公共边界、重要数据结构和函数返回值声明清晰的类型；条件分支使用花括号。
- Bash 中始终正确引用变量，函数内变量使用 `local`，显式检查命令是否成功，并避免依赖不明显的 shell 展开或全局状态。
- 不做无关重构；但修改某段代码时，应删除该范围内已经不再使用的分支、变量和注释。
- 注释解释“为什么”和约束条件，不复述代码本身。跨 Bash 与 TypeScript 维护的镜像逻辑必须同步修改。

## 变更要求

- 新功能应直接替换旧设计，不要并行维护 legacy 路径。
- 配置项、状态值或命令行为发生变化时，同步更新 `README.md`。
- 行为变更必须添加或更新测试，测试名称应明确描述场景和预期结果。
- 修复问题时优先解决根因，不使用静默吞错、无边界重试或仅针对单个样例的补丁。
- 除非任务明确要求，否则不要引入新的运行时依赖或测试框架。

## 项目结构

- `agents_session_manager.tmux`：插件入口和 tmux 配置初始化。
- `scripts/`：启动、选择器、状态管理及共享 Bash 逻辑。
- `extensions/tmux-state.ts`：Pi 状态上报扩展。
- `tests/run.sh`：主要 Bash 测试套件。
- `tests/perf_smoke.sh`：性能冒烟测试。

## 验证

完成代码修改后，至少运行：

```bash
bash tests/run.sh
```

涉及选择器、进程扫描、状态统计或性能敏感路径时，再运行：

```bash
bash tests/perf_smoke.sh
```

如果环境中安装了 ShellCheck，修改 Bash 文件后运行：

```bash
shellcheck agents_session_manager.tmux scripts/*.sh tests/*.sh
```

提交结果前确认测试通过，并检查文档、实现和测试描述保持一致。
